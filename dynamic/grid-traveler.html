Memoization in Dynamic Programming: Grid Traveler

Memoization is a top-down dynamic-programming technique that speeds up recursive algorithms by caching subproblem results. The Grid Traveler problem asks: “How many ways can you move from the top-left to the bottom-right of an m × n grid if you may only move right or down?” (No diagonal moves.)

How It Works

Define the subproblem T(m,n) = number of ways to reach (1,1) from (m,n) using only up/left (equivalently, from (1,1) to (m,n) using right/down). The recurrence:

T(m, n) = T(m-1, n) + T(m, n-1)
Base cases: T(1,1) = 1;  T(m,0) = 0;  T(0,n) = 0

With memoization, we store each computed T(m,n) so repeated calls return in O(1).

Step-by-Step (Top-Down)

  1. State: Subproblem is the pair (m,n).
  2. Check cache: If (m,n) is in the memo table, return it immediately.
  3. Base cases: Return 1 for (1,1); return 0 if m=0 or n=0.
  4. Recurse: Compute T(m-1,n) and T(m,n-1), add them.
  5. Save & return: Store T(m,n) in the memo and return it.

Example (3 × 3)

Evaluate T(3,3):

  • T(3,3) = T(2,3) + T(3,2)
  • T(2,3) = T(1,3) + T(2,2) and T(3,2) = T(2,2) + T(3,1)
  • Subproblems like T(2,2) appear multiple times — memoization caches them once.

The result is 6 distinct paths for a 3×3 grid.

Reference Implementation (JavaScript)

Memo table key is the pair (m,n) (you can also normalize as m,n string).


      // Top-down DP with memoization for Grid Traveler
      function gridTraveler(m, n, memo = new Map()) {
        const key = `${m},${n}`;                // key for (m,n)
        if (memo.has(key)) return memo.get(key);
        if (m === 1 && n === 1) return 1;       // base success
        if (m === 0 || n === 0) return 0;       // base failure
        const ways = gridTraveler(m - 1, n, memo) + gridTraveler(m, n - 1, memo);
        memo.set(key, ways);
        return ways;
      }

      // Example: gridTraveler(3, 3) === 6

Blocked Cells (Optional)

If certain cells are blocked, treat those states as 0:


      function gridTravelerBlocked(m, n, blocked, memo = new Map()) {
        const key = `${m},${n}`;
        if (memo.has(key)) return memo.get(key);
        if (m === 1 && n === 1) return blocked.has('1,1') ? 0 : 1;
        if (m === 0 || n === 0) return 0;
        if (blocked.has(`${m},${n}`)) return 0;
        const ways = gridTravelerBlocked(m - 1, n, blocked, memo)
                  + gridTravelerBlocked(m, n - 1, blocked, memo);
        memo.set(key, ways);
        return ways;
      }

Characteristics

  • Technique: Top-down recursion + memo table (cache).
  • Overlapping Subproblems: Same (m,n) requested multiple times.
  • Optimal Substructure: T(m,n) depends only on T(m-1,n) and T(m,n-1).
  • Time Complexity: O(m·n) (each state computed once).
  • Space Complexity: O(m·n) for the memo plus recursion depth.

Memoization converts the naive exponential recursion into efficient linear-in-states performance, making the Grid Traveler a perfect example of top-down dynamic programming.
