Explanation of Greedy Programming with Activity Selection Algorithm

What is Greedy Programming?

Greedy programming is an algorithmic paradigm that builds up a solution piece by piece, always choosing the next piece that offers the most immediate benefit. It is an approach used to solve optimization problems. The core idea is to make a locally optimal choice at each stage with the hope of finding a globally optimal solution.

Greedy algorithms are generally simple to implement and very efficient in terms of time complexity. However, they do not always guarantee a globally optimal solution for every problem. The Activity Selection Problem is one classic example where the greedy approach *does* yield the optimal solution.

The Activity Selection Problem

The problem is defined as follows: Given a set of n activities, each with a specific start time si and finish time fi, the goal is to select the maximum number of non-overlapping activities.

Two activities, i and j, are non-overlapping if their time intervals si, fiand sj, fi do not intersect. This means either fi <= sj or fj <= si.

The Greedy Approach for Activity Selection

The key to solving this problem efficiently is to choose activities based on their finish times. The most effective greedy strategy is to always pick the activity that finishes earliest among those that are compatible with the activities already selected.

The Algorithm Steps:

  1. Sort: Sort all activities in ascending order of their finish times. This is the crucial greedy choice criteria.
  2. Select the First: Select the first activity (which has the earliest finish time).
  3. Iterate and Filter: Iterate through the remaining sorted activities. For each activity, check if its start time is greater than or equal to the finish time of the last selected activity.
  4. Add Compatible: If they are compatible (non-overlapping), select the current activity and update the last selected activity's finish time.

Why Does This Work?

By picking the activity that finishes soonest, we maximize the available time remaining for subsequent activities. If we picked an activity that finished later, even if it started early, it would limit the number of other potential activities that could fit after it.

Execution:

  • Step 1: Select A1 (finishes at 2). The last finish time is now 2.
  • Step 2: Consider A2 (starts at 3). 3 >= 2, so it is compatible. Select A2 (finishes at 4). The last finish time is now 4.
  • Step 3: Consider A3 (starts at 0). 0 < 4, so it overlaps with A2. Reject A3.
  • Step 4: Consider A4 (starts at 5). 5 >= 4, so it is compatible. Select A4 (finishes at 7). The last finish time is now 7.
  • Step 5: Consider A5 (starts at 8). 8 >= 7, so it is compatible. Select A5 (finishes at 9). The last finish time is now 9.
  • Step 6: Consider A6 (starts at 5). 5 < 9, so it overlaps with A5. Reject A6.

The maximum set of non-overlapping activities is {A1, A2, A4, A5}.

Algorithm in Pseudocode

Algorithm GreedyActivitySelection(S, F): Input: S (start times), F (finish times) sorted by finish times. Output: A (a set of compatible activities). n = length of S A = {activity 1} // Select the first activity k = 1 // Index of the last selected activity for i from 2 to n: if S[i] >= F[k]: A = A U {activity i} k = i return A
