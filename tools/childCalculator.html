Understanding Binary Heap Child Index Calculation

In a binary heap stored as an array, each node at index i has:

  • Left child index: 2 * i + 1
  • Right child index: 2 * i + 2

Why These Formulas Work

Binary heaps are complete binary trees, and when stored in arrays, their structure follows a predictable pattern:

  • The root node is at index 0.
  • Each level fills left to right, so children of node i are placed immediately after all previous nodes.
  • Thus, the left child is at 2 * i + 1 and the right child is at 2 * i + 2.

Example

Given a node at index 3:

  • Left child index = 2 * 3 + 1 = 7
  • Right child index = 2 * 3 + 2 = 8

This indexing makes traversal and heap operations efficient without needing pointers or tree nodes.
