Backtracking Programming with N-Queens Algorithm

Backtracking is a problem-solving algorithmic technique that explores all possible solutions by building them step by step and then abandoning (backtracking from) any path that fails to satisfy the problem's constraints. It's a form of informed brute-force search that is well-suited to the N-Queens problem because it prunes unviable solution branches, significantly reducing the search space.

The N-Queens Problem

The N-Queens problem challenges you to place $N$ chess queens on an $N \times N$ chessboard so that no two queens attack each other. A queen can attack horizontally, vertically, and diagonally. Therefore, a valid solution must ensure that no two queens share the same:

  • Row
  • Column
  • Main diagonal (top-left to bottom-right)
  • Anti-diagonal (top-right to bottom-left)

Backtracking Algorithm for N-Queens

The standard backtracking approach for the N-Queens problem involves placing one queen per row, one by one, from the first row to the last.

The algorithm uses a recursive function that attempts to place a queen in each column of the current row: 

function solveNQueens(row, N, board):
    if row == N:
        // All queens placed successfully, add board to solutions
        printSolution(board)
        return

    for col from 0 to N-1:
        if isSafe(row, col, board):
            // Place queen
            board[row][col] = 'Q'
            // Recurse to next row
            solveNQueens(row + 1, N, board)
            // Backtrack: remove queen and try next column
            board[row][col] = '.'

Algorithm Steps Explained

  1. Start at the first row: The process begins by trying to place a queen in a column of the first row.
  2. Check for safety: For each potential position (row, col), the algorithm checks if placing a queen there is safe from previous queens (in previous rows).
    • No other queen in the same column.
    • No other queen in the same main diagonal (row - col is constant).
    • No other queen in the same anti-diagonal (row + col is constant).
  3. Place and Recurse: If the position is safe, the queen is placed, and the function recursively calls itself for the next row. This "explores" the current valid path.
  4. Backtrack: If the recursive call returns (meaning no safe position was found in a subsequent row), the current queen is removed from its position, and the algorithm tries the next column in the current row. This is the "backtracking" step.
  5. Repeat: Steps 2-4 repeat until a solution is found (all $N$ queens placed) or all possibilities have been explored. Once a solution is found, the function continues to backtrack to find all other distinct solutions.
