Breadth-First Search (BFS)

Breadth-First Search (BFS) is a systematic graph traversal algorithm that explores vertices level by level. Starting from a chosen source node, BFS visits all its immediate neighbors first, then moves outward to their unvisited neighbors. It guarantees the shortest path (in number of edges) in an unweighted graph.

How It Works

The process follows a clear sequence:

  1. Initialize: Choose a starting node and mark it as visited.
  2. Use a Queue: Enqueue the start node into a queue (FIFO order).
  3. Process: Repeat until the queue is empty:
    • Dequeue a node from the front.
    • Visit all its unvisited neighbors.
    • Mark them as visited and enqueue them.
  4. End: When the queue is empty, all reachable nodes have been explored.

Example

Consider the following simple graph (represented as adjacency lists):

A: B, C
B: D, E
C: F
D: —
E: —
F: —

If we start BFS from node A:

  • Begin with queue = [A]; mark A as visited.
  • Dequeue A → visit its neighbors B and C → enqueue them. Queue = [B, C]
  • Dequeue B → visit neighbors D and E → enqueue them. Queue = [C, D, E]
  • Dequeue C → visit neighbor F → enqueue it. Queue = [D, E, F]
  • Continue until the queue is empty. The visit order is A → B → C → D → E → F.

Characteristics

  • Traversal Order: Explores nodes in layers—closer nodes first, deeper ones later.
  • Shortest Path: Ensures shortest path in unweighted graphs.
  • Data Structure: Uses a Queue for node management.
  • Applicability: Works on trees and general graphs (directed or undirected).
  • Time Complexity: O(V + E), where V = vertices and E = edges.
  • Space Complexity: O(V) for storing visited and queued nodes.

BFS is essential in applications like shortest-path finding, social-network analysis, and web-crawling algorithms, thanks to its organized, level-wise exploration.
